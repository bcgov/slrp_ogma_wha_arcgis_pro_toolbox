# Check In Dataset

## What It Does

**Check In Dataset** automates the multi-step dataset check-in workflow for OGMA, LU, SLRP, and WHA submissions. It runs all check-in steps in sequence from a single ArcGIS Pro tool dialog, replacing the manual copy-and-check procedure.

The tool is **read-only on the source data throughout** — it never edits or locks the Returned FGDB. It copies data out; it never writes data back in.

---

## When to Use It

Use this tool after a dataset has been returned by the submitter and you are ready to:

1. Confirm all required files are present in the update directory.
2. Run Attribute QA/QC before accepting the submission.
3. Review the topology report.
4. Promote the Returned FGDB to `UpdateManagement\<Type>\CurrentUpdate`.
5. Place the QA and topology reports in the Update Emails folder for the request.

---

## Prerequisites

Before running the tool, the update directory must contain:

| File | Description |
|---|---|
| `<DatasetName>.gdb` | The Returned FGDB (exactly one `.gdb` per update directory) |
| `<name>_attribute_check_<date>.txt` | QA/QC report from a previous run (optional; a new one is generated by this tool) |
| `topology_report.csv` | Topology error report exported from ArcGIS Pro (UTF-16 LE encoding) |

The tool checks for all three and reports what is present or missing before proceeding. Missing files generate warnings but do not stop the tool (except a missing Returned FGDB, which aborts).

The GDB name must contain one of these substrings for the tool to auto-detect the dataset type:

| GDB name contains | Type detected |
|---|---|
| `old_growth_management_area` | OGMA |
| `landscape_unit` | LU |
| `slrp_planning` | SLRP |
| `wildlife_habitat_area` | WHA |

---

## Parameters

| # | Parameter | Type | Description |
|---|---|---|---|
| 1 | **Update Work Area Directory** | Folder | The `UpdateWorkArea\<Type>` folder containing the Returned FGDB and report files. Defaults to the OGMA path — change as needed. |
| 2 | **Input Feature Class (inside Returned FGDB)** | Feature Layer | The feature class inside the Returned FGDB to run QA against. Add it to the map first, or browse to it. |
| 3 | **Master Feature Class (for Attribute QA)** | Feature Layer | The master/replica feature class used as the reference for QA field and domain checking. |
| 4 | **Update Email Folder for This Request** | Folder | The existing `Update_Emails\<RequestFolder>` folder where QA and topology reports are copied. |

---

## Workflow Steps

The tool runs 6 steps in order. A progress bar tracks each step.

### Step 1 — File Checklist
Scans the **Update Work Area Directory** and reports whether the Returned FGDB, QA/QC report(s), and `topology_report.csv` are present or missing. No files are modified.

### Step 2 — Derive Dataset Type
Reads the name of the Returned GDB and matches it to a dataset type (OGMA / LU / SLRP / WHA). **If no match is found, the tool aborts here.** The type is used to determine the correct destination path under `UpdateManagement`.

### Step 3 — Attribute QA/QC *(blocking gate)*
Runs the Attribute QA tool (`attribute_qa_v8`) against the Input Feature Class, comparing it to the Master Feature Class. Generates `.txt`, `.json`, and `.html` report files alongside the input dataset.

**If QA raises an error, the tool aborts and the copy steps do not run.** Fix the reported issues and re-run.

### Step 4 — Topology Report Display
Reads `topology_report.csv` from the update directory and prints every row to the tool messages panel. **This step is informational only — the tool does not pass or fail based on topology errors.** The user determines whether the topology is acceptable.

### Step 5 — Copy Returned FGDB
Copies the entire Returned FGDB to:

```
\\spatialfiles3.bcgov\slrp\UpdateManagement\<TypeFolder>\CurrentUpdate\<GDBName>.gdb
```

Uses `arcpy.management.Copy()`. If a GDB with the same name already exists at the destination, it is overwritten. The source FGDB is never modified.

### Step 6 — Copy Reports to Update Emails Folder
Copies the following files directly into the **Update Email Folder**:

- `<name>_attribute_check_<date>.txt`
- `<name>_attribute_check_<date>.json`
- `<name>_attribute_check_<date>.html`
- `topology_report.csv`

Files that do not exist are skipped with a warning.

---

## Abort Conditions

The tool stops early and does not run the copy steps if:

- The update directory contains zero or more than one `.gdb`
- The GDB name does not match any known dataset type substring
- Attribute QA/QC raises an exception

In all cases the tool message panel explains what went wrong.

## Output Locations

| What | Where |
|---|---|
| Returned FGDB | `\\spatialfiles3.bcgov\slrp\UpdateManagement\<TypeFolder>\CurrentUpdate\` |
| QA report (.txt / .json / .html) | The Update Email Folder you specified |
| Topology report (.csv) | The Update Email Folder you specified |
